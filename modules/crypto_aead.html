Module crypto_aead

Low-level Lua interface to the Linux Kernel Crypto API for AEAD (Authenticated Encryption with Associated Data) ciphers.

This module provides a new function to create AEAD transform objects, which can then be used for encryption and decryption.

Class crypto_aead

AEAD object methods. These methods are available on AEAD objects created by crypto_aead.new().
crypto_aead:authsize ()
Gets the current authentication tag size for the AEAD transform. This is the value set by setauthsize or the algorithm's default.

Returns:

    integer The authentication tag size in bytes.
crypto_aead:decrypt (iv, combined_data, aad_len)
Decrypts data using the AEAD transform. The IV (nonce) and AAD must match those used during encryption.

Parameters:

  • iv string The Initialization Vector (nonce). Its length must match ivsize().
  • combined_data string A string containing AAD (Additional Authenticated Data) concatenated with the ciphertext and tag (format: AAD || Ciphertext || Tag).
  • aad_len integer The length of the AAD part in combined_data.

Returns:

    string The decrypted data, formatted as (AAD || Plaintext).

Raises:

Error on decryption failure (e.g., authentication error - EBADMSG), incorrect IV length, input data too short, or allocation issues.
crypto_aead:encrypt (iv, combined_data, aad_len)
Encrypts data using the AEAD transform. The IV (nonce) must be unique for each encryption operation with the same key.

Parameters:

  • iv string The Initialization Vector (nonce). Its length must match ivsize().
  • combined_data string A string containing AAD (Additional Authenticated Data) concatenated with the plaintext (format: AAD || Plaintext).
  • aad_len integer The length of the AAD part in combined_data.

Returns:

    string The encrypted data, formatted as (AAD || Ciphertext || Tag).

Raises:

Error on encryption failure, incorrect IV length, or allocation issues.
crypto_aead:ivsize ()
Gets the required initialization vector (IV) size for the AEAD transform.

Returns:

    integer The IV size in bytes.
crypto_aead:new (algname)
Creates a new AEAD object. This is the constructor function for the crypto_aead module.

Parameters:

  • algname string The name of the AEAD algorithm (e.g., "gcm(aes)", "ccm(aes)").

Returns:

    crypto_aead The new AEAD object.

Raises:

Error if the TFM object or kernel request cannot be allocated/initialized.

Usage:

    local aead = require("crypto_aead")
local cipher = aead.new("gcm(aes)")
crypto_aead:setauthsize (tagsize)
Sets the authentication tag size for the AEAD transform.

Parameters:

  • tagsize integer The desired authentication tag size in bytes.

Raises:

Error if setting the authsize fails (e.g., unsupported size).
crypto_aead:setkey (key)
Sets the encryption key for the AEAD transform.

Parameters:

  • key string The encryption key.

Raises:

Error if setting the key fails (e.g., invalid key length for the algorithm).
